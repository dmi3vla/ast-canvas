# Шаблон задачи агенту

Заполняется постановщиком для **одного** этапа из `IMPLEMENTATION_ROADMAP.md`.

```text
Ты выполняешь только фазу <N>, этап <N.M> из docs/IMPLEMENTATION_ROADMAP.md.

Продуктовая цель:
<какой пользовательский вопрос решает этот этап>

Контрольный сценарий:
<например: pnpm dev → Open Folder → fixtures/mini-project>

Наблюдаемый результат:
<одно проверяемое предложение о том, что видно и что работает>

Визуальный контракт:
- референс: docs/proto1.png
- сравниваемое состояние: <empty/map/selected/codemap/source>
- важные признаки: <3–5 конкретных признаков>

Границы:
- менять можно: <файлы/слои>
- не менять: <явный список>
- не расширять scope: <явный список>

Критерии приёмки:
1. <поведение>
2. <визуальный критерий>
3. <данные/регрессия>

Доказательство готовности:
- screenshot: <путь>
- команды: <typecheck/test/build или конкретные тесты>
- ручные шаги: <1–3 шага>

Порядок работы:
1. Сначала прочитай существующий код и референс.
2. Сделай минимальную правку для результата.
3. Проверь контрольный сценарий.
4. Не начинай следующий этап.

Финальный отчёт:
- что изменено;
- screenshot/результат проверки;
- что осталось расхождением с референсом;
- вопросы, требующие решения постановщика.
```

## Пример краткой заполненной задачи

```text
Фаза 1, этап 1.2 — единая визуальная система canvas.

Сценарий: загрузить golden semantic map fixture.
Результат: header, area и file cards визуально различимы; карта читается
при первом fit view и не содержит debug labels.
Референс: docs/proto1.png, состояние map.
Не менять: semantic pipeline, IPC, DepGraph, RIGHT navigation.
Приёмка: screenshot docs/render-phase1-2.png; pnpm typecheck; selection работает.
```
